# 《面向高校毕业生的中国就业态势感知与职业决策支持平台》毕业设计方案

## 1. 项目背景与研究意义

高校毕业生就业选择通常受到城市机会、行业景气度、薪资预期、学历门槛、经验要求和个人技能匹配度等因素共同影响。传统就业指导主要依赖人工经验、静态报告或单一招聘网站查询，难以及时反映全国招聘市场的区域差异、岗位结构变化和技能需求趋势。

本项目基于 Kaggle 中国招聘岗位数据集 `China Jobs Data` 与 `Job Posting Data in China`，整合岗位名称、公司名称、城市、省份、薪资范围、学历要求、经验要求、行业类别、专业要求、公司规模、公司性质、发布时间、岗位描述、经纬度等字段，构建一个面向高校毕业生的就业态势感知与职业决策支持平台。

项目意义体现在三方面：

1. 对毕业生：帮助学生理解目标城市、目标行业、岗位薪资和技能要求，降低盲目投递成本。
2. 对就业指导老师：提供可解释的数据指标和推荐结果，辅助开展精准就业指导。
3. 对高校管理部门：通过动态大屏呈现全国就业热度、城市吸引力和岗位变化趋势，为就业管理和专业建设提供依据。

该系统不是普通统计系统，而是将招聘大数据处理、机器学习预测、技能文本挖掘、职业推荐和动态可视化驾驶舱结合起来，形成可展示、可解释、可扩展的就业决策支持平台。

## 2. 系统需求分析

### 2.1 用户角色

- 高校毕业生：关注岗位方向、城市选择、薪资水平、技能差距和投递建议。
- 就业指导老师：关注不同专业学生适配岗位、城市机会、技能短板和指导报告。
- 高校就业管理部门：关注全国、省市、行业层面的就业态势和趋势变化。
- 系统管理员：负责数据导入、模型训练、指标刷新和用户权限维护。

### 2.2 功能需求

1. 全国就业态势感知  
   统计全国各省、市岗位数量、平均薪资、热门行业、学历要求、经验门槛和就业热度指数。

2. 城市就业吸引力评估  
   计算城市就业吸引力指数，综合岗位数量、平均薪资、应届生友好度、行业丰富度、学历门槛和经验要求，并形成城市排行榜。

3. 岗位薪资预测  
   用户选择城市、行业、学历、经验、公司规模、岗位类别后，系统预测岗位薪资区间和置信说明。

4. 技能需求挖掘  
   对岗位名称和岗位描述进行中文分词、TF-IDF、TextRank 和技能词典匹配，生成技能热度、技能趋势和岗位技能画像。

5. 高校毕业生职业推荐  
   用户输入专业、学历、技能、期望城市和期望行业，系统推荐岗位方向、目标城市、岗位类别、匹配度和技能提升建议。

6. 动态可视化大屏  
   通过 ECharts 中国地图、飞线、涟漪热点、翻牌指标、滚动榜单、趋势播放和实时岗位流展示就业市场态势。

7. 多页面业务工作台  
   在大屏之外提供城市评估、薪资预测、技能挖掘、职业推荐和数据治理页面，使系统具备可操作的业务功能，而不是只有展示型大屏。

### 2.3 非功能需求

- 可用性：前端大屏适配 1920x1080 分辨率，核心数据首屏可见。
- 可解释性：指数、预测和推荐都需要展示核心影响因素。
- 可扩展性：后端接口按模块拆分，便于后续接入真实招聘平台数据。
- 性能：大屏接口返回聚合数据，避免前端一次加载全量岗位明细。
- 安全性：管理端接口预留鉴权，用户输入参数进行校验。

## 3. 系统功能模块设计

系统划分为六个核心模块：

1. 数据采集与治理模块  
   负责导入 Kaggle CSV 数据，完成字段统一、薪资解析、城市省份标准化、学历经验标准化、经纬度补全和异常数据清洗。

2. 就业态势分析模块  
   负责省市岗位规模、薪资、行业、学历、经验和岗位增长趋势分析，形成全国就业态势指标。

3. 城市吸引力评估模块  
   按城市聚合岗位数量、平均薪资、应届生友好度、行业丰富度、学历门槛、经验门槛等指标，输出城市排名和指标解释。

4. 智能预测与挖掘模块  
   包含薪资预测模型、技能需求热度模型、岗位趋势分析模型。薪资模型可采用 RandomForest、XGBoost 或 LightGBM；技能挖掘采用 jieba、TF-IDF、TextRank 和词频统计。

5. 职业推荐模块  
   基于用户画像与岗位标签进行多因子匹配，输出岗位方向、城市建议、技能补齐建议和匹配度。

6. 动态可视化大屏模块  
   包括全国地图、核心指标、热门城市和行业、薪资学历经验分析、技能词云、岗位发布滚动列表、城市吸引力排行和 12 个月趋势播放。

7. 业务工作台模块  
   包括城市评估、薪资预测、技能挖掘、职业推荐和数据治理五个页面。城市评估页面展示重点城市卡片和省份覆盖表；薪资预测页面提供预测表单与解释因子；技能挖掘页面展示技能热度排行和提升建议；职业推荐页面输出毕业生岗位方向矩阵；数据治理页面展示从 Kaggle CSV 到指标库和模型服务的处理链路。

## 4. 数据库表结构设计

数据库采用 MySQL，主要表如下：

### 4.1 原始岗位表 `job_post`

保存清洗后的岗位明细，包括岗位名称、公司、城市、省份、薪资、学历、经验、行业、专业、描述和经纬度。

### 4.2 城市指标表 `city_employment_index`

保存城市维度的岗位数量、平均薪资、行业丰富度、应届生友好度、学历门槛、经验门槛、吸引力指数和排名。

### 4.3 省份指标表 `province_employment_index`

保存省份就业热度、岗位数量、平均薪资、应届生友好度、热门行业和趋势变化。

### 4.4 技能关键词表 `skill_keyword`

保存技能名称、岗位类别、行业、词频、TF-IDF 权重、TextRank 权重、热度指数和统计月份。

### 4.5 模型预测记录表 `salary_prediction_record`

保存用户预测输入、模型输出薪资和模型版本，用于后续效果评估。

### 4.6 用户职业画像表 `graduate_profile`

保存毕业生专业、学历、技能、期望城市、期望行业和职业偏好。

### 4.7 职业推荐结果表 `career_recommendation`

保存推荐岗位方向、推荐城市、推荐行业、匹配度、推荐理由和技能提升建议。

完整 SQL 位于 `database/schema.sql`。

## 5. 数据处理流程

1. 数据导入  
   从 Kaggle 下载 `China Jobs Data` 与 `Job Posting Data in China`，将 CSV 文件放入 `data-processing/data/`。

2. 字段映射  
   统一不同数据集字段名，例如 `job_title/title` 映射为岗位名称，`city/work_city` 映射为工作城市。

3. 薪资解析  
   将 `10k-20k/月`、`15-25K`、`20万/年` 等文本解析为最低薪资、最高薪资和平均月薪。

4. 地域标准化  
   对城市、省份名称进行标准化，补全省份、经纬度和城市等级信息。当前真实快照覆盖大陆 31 个省级区域，香港有少量 Kaggle 样本；澳门、台湾暂无同口径岗位样本，应展示“样本不足/置信度较低”而不是直接留空或用虚构数据补齐。

5. 学历经验标准化  
   将学历映射为数值门槛：不限、大专、本科、硕士、博士；经验映射为不限、应届、1-3 年、3-5 年、5 年以上。

6. 文本处理  
   合并岗位名称和岗位描述，使用 jieba 分词、停用词过滤、技能词典匹配、TF-IDF 和 TextRank 提取关键词。

7. 指标聚合  
   按省份、城市、行业、月份聚合岗位数量、平均薪资、增长率、技能热度和学历经验分布。

8. 模型训练  
   构建薪资预测训练集，使用类别特征编码和随机森林、XGBoost 或 LightGBM 训练模型，并保存模型文件。

9. 数据入库  
   将清洗岗位、聚合指标、技能结果和模型结果写入 MySQL，供后端接口读取。

## 6. 核心算法设计

### 6.1 城市就业吸引力指数

城市就业吸引力指数用于评价毕业生选择城市的综合价值。设：

- `J`：岗位数量标准化分
- `S`：平均薪资标准化分
- `F`：应届生友好度
- `D`：行业丰富度标准化分
- `E`：学历门槛友好度
- `X`：经验门槛友好度

公式：

```text
CityScore = 0.25J + 0.25S + 0.20F + 0.15D + 0.08E + 0.07X
```

其中岗位数量和薪资体现机会规模与回报，应届生友好度体现毕业生适配性，行业丰富度体现职业选择空间，学历和经验门槛体现进入难度。

### 6.2 应届生友好度指数

应届生友好度指数综合不限经验岗位比例、本科可投比例、平均起薪和岗位增长率：

```text
FreshScore = 0.35 * NoExperienceRatio
           + 0.25 * BachelorAvailableRatio
           + 0.25 * NormalizedEntrySalary
           + 0.15 * JobGrowthRateScore
```

该指标适合高校就业指导场景，能解释某城市是否适合毕业生第一份工作。

### 6.3 薪资预测模型

输入特征：

- 城市、省份、行业、岗位类别
- 学历要求、经验要求
- 公司规模、公司性质
- 技能关键词数量和技能类别
- 发布时间月份

目标变量：

- 平均月薪 `salary_avg`
- 或薪资区间标签 `low / medium / high`

模型选择：

- 本科毕设基础版：RandomForestRegressor，部署简单、解释性较强。
- 增强版：XGBoost 或 LightGBM，提高非线性特征拟合能力。

评价指标：

- MAE：平均绝对误差
- RMSE：均方根误差
- R2：模型解释能力

### 6.4 技能需求热度模型

处理步骤：

1. 使用 jieba 对岗位名称和描述分词。
2. 使用停用词表过滤无效词。
3. 通过技能词典识别生产、管理、销售、机械、沟通、安全、质量、会计、电气、Excel、Python、SQL 等岗位关键词。
4. 计算词频、TF-IDF 权重和 TextRank 权重。
5. 结合岗位数量和月份趋势计算技能热度：

```text
SkillHot = 0.45 * FrequencyScore
         + 0.30 * TfidfScore
         + 0.15 * TextRankScore
         + 0.10 * TrendScore
```

### 6.5 职业推荐模型

推荐模型基于用户画像和岗位标签进行多因子匹配：

```text
MatchScore = 0.30 * SkillMatch
           + 0.20 * MajorMatch
           + 0.15 * EducationFit
           + 0.15 * CityPreference
           + 0.10 * IndustryPreference
           + 0.10 * SalaryPotential
```

输出内容包括岗位方向、推荐城市、推荐行业、匹配度、推荐理由和技能提升建议。该模型适合本科项目实现，因为可先用规则加权完成，再逐步加入协同过滤或向量相似度。

## 7. 动态可视化大屏设计方案

大屏定位为“就业态势感知驾驶舱”，采用深色科技风，整体分为左、中、右、底四个区域，适配 1920x1080 展示。

本项目的前端不是单一大屏。顶部导航提供“态势大屏、城市评估、薪资预测、技能挖掘、职业推荐、数据治理”六个入口，其中大屏负责全局态势展示，其他页面承载具体分析和决策支持功能。

### 7.1 中央区域：动态中国地图

- 使用 ECharts 中国地图作为核心画布。
- 省份颜色根据就业热度指数映射。
- 自动轮播省份高亮，并同步更新右侧省份详情。
- 鼠标悬停显示岗位数量、平均薪资、应届生友好度和热门行业。
- 点击省份联动右侧详情面板。
- 使用 effectScatter 展示热点城市涟漪。
- 使用 lines 展示省会城市之间的就业流向飞线。

### 7.2 左侧区域：核心指标与排行

- 全国岗位总数、平均薪资、覆盖城市数、应届生友好度采用翻牌数字动画。
- 热门城市 TOP10 自动滚动。
- 热门行业 TOP10 自动滚动。
- 排名条随数据变化平滑过渡。

### 7.3 右侧区域：薪资、学历与技能分析

- 薪资区间分布使用动态柱状图。
- 学历要求和经验要求使用环形图或玫瑰图。
- 技能关键词使用词云或自定义流动标签。
- 图表按固定时间切换或刷新，避免静态展示。

### 7.4 底部区域：实时就业动态

- 岗位发布列表持续向上滚动。
- 城市就业吸引力排行榜横向展示。
- 近 12 个月岗位趋势使用折线图，并通过时间轴或数据窗口动态播放。

## 8. 前后端系统架构设计

### 8.1 总体架构

```text
Kaggle 数据集
    ↓
Python/Pandas 数据清洗与模型训练
    ↓
MySQL 指标库与岗位库
    ↓
Python FastAPI + SQLAlchemy 后端接口
    ↓
Vue3 + TypeScript + ECharts 多模块前端平台
```

### 8.2 后端架构

- Router 层：使用 FastAPI APIRouter 提供 REST API。
- Service 层：封装就业态势、城市指数、薪资预测和职业推荐逻辑。
- Repository 层：预留 SQLAlchemy 查询接口，用于访问 MySQL。
- Schema 层：使用 Pydantic 定义请求参数、响应结构和校验规则。

核心接口：

- `GET /api/dashboard/overview`
- `GET /api/dashboard/provinces`
- `GET /api/dashboard/cities/ranking`
- `GET /api/dashboard/analysis`
- `GET /api/dashboard/skills`
- `GET /api/dashboard/live-jobs`
- `POST /api/predict/salary`
- `POST /api/recommend/career`

### 8.3 前端架构

- Vue3 + TypeScript + Vite
- ECharts 负责地图、柱状图、折线图、环形图和飞线。
- Element Plus 负责表单、按钮、下拉选择等业务交互组件。
- CSS 动效负责翻牌数字、滚动榜单、背景网格、粒子、流光和热点涟漪。
- 顶部模块导航负责在态势大屏、城市评估、薪资预测、技能挖掘、职业推荐和数据治理页面之间切换。

## 10. 项目创新点与可行性分析

### 10.1 创新点

1. 将招聘岗位大数据与高校就业指导场景结合，面向毕业生做职业决策支持。
2. 构建城市就业吸引力指数和应届生友好度指数，使城市选择具备可解释依据。
3. 引入岗位薪资预测模型，帮助毕业生形成合理薪资预期。
4. 基于中文岗位描述挖掘技能需求，输出技能热度与提升建议。
5. 使用动态可视化大屏呈现就业态势，适合答辩演示和管理决策。
6. 推荐结果不仅给岗位，还给城市、行业和技能补齐路径。

### 10.2 可行性

- 数据可行：Kaggle 中国招聘岗位数据字段完整，适合清洗、聚合和建模。
- 技术可行：FastAPI、Vue3、ECharts、MySQL、Pandas 和 Scikit-learn 都是成熟技术。
- 算法可行：指数模型、随机森林薪资预测、TF-IDF 和规则匹配推荐适合本科毕业设计实现。
- 展示可行：前端可使用真实快照作为接口兜底数据，并通过后端接口展示动态大屏。
- 扩展可行：后续可接入实时招聘 API、学校就业数据、学生画像和投递反馈数据。

## 12. 大屏动态效果实现方案

1. 数字翻牌  
   使用 Vue 组合式函数对指标数值进行补间动画，结合 CSS 数字卡片实现翻牌感。

2. 省份轮播高亮  
   使用 `setInterval` 定时调用 ECharts `dispatchAction`，依次触发 `highlight`、`downplay` 和 `showTip`。

3. 城市涟漪热点  
   使用 ECharts `effectScatter`，通过 `rippleEffect` 表现城市岗位热度扩散。

4. 省会飞线  
   使用 ECharts `lines` 系列，设置 `effect.symbol` 和 `period` 表示岗位流向。

5. 排行榜自动滚动  
   使用 CSS `transform: translateY()` 和循环数组实现无缝滚动。

6. 实时岗位列表持续滚动  
   使用重复列表和 CSS keyframes，鼠标悬停时暂停，便于查看。

7. 趋势图动态播放  
   定时更新折线图数据窗口或当前月份标记，通过 ECharts `setOption` 平滑过渡。

8. 深色科技背景  
   使用 CSS 网格背景、扫描光、低透明粒子层和角标线条，避免页面像静态报表。

9. 平滑图表切换  
   ECharts 开启动画，设置 `animationDurationUpdate` 和 `animationEasingUpdate`，切换数据时保留上下文。

10. 1920x1080 适配  
   大屏容器按 `16 / 9` 设计，使用 CSS Grid 固定四区比例，并通过 `minmax` 和媒体查询兼容较小屏幕。
